# ADR 0006 — Distribution: unsigned ad-hoc build via a private Homebrew cask tap

- **Status:** Accepted (2026-06-07)

## Context

corti is a menu-bar `.app` that captures microphone + system audio. ADR 0001 said corti "gets its own
release pipeline"; ADR 0002 fixed the platform to Apple Silicon, latest macOS only. We now need to actually
ship it: a native build and an install channel. The hard question is **code signing**.

Three macOS gates are easy to conflate:

1. **arm64 execution** — Apple Silicon refuses to run native code with *no* signature. Tauri's default
   **ad-hoc** signing (`codesign -s -`) satisfies this; corti already runs locally as an ad-hoc bundle.
2. **Gatekeeper / quarantine** — anything downloaded with the `com.apple.quarantine` xattr must be
   Developer-ID-signed **and notarized**, or macOS blocks first launch. As of macOS Sequoia the old
   right-click→Open bypass is gone; the user must visit System Settings → Privacy & Security → "Open
   Anyway" + admin auth. **Developer-ID-signed-but-not-notarized is blocked exactly like unsigned** —
   signing only helps Gatekeeper if you *also* notarize.
3. **TCC** (the mic + System-Audio-Recording grants corti needs) — keys on the bundle's code-signing
   identity. For ad-hoc builds that identity is the cdhash, which changes on every build, so each new
   release is treated as a new app and re-prompts. A stable Developer ID identity persists grants across
   updates.

Notarization (gate 2) requires a paid Apple Developer Program membership ($99/yr), a Developer ID cert in
CI, Hardened Runtime entitlements, and a notarytool submit/staple step.

Homebrew context (mid-2026): Homebrew 5.0 (Nov 2025) **removed the `--no-quarantine` flag**, so casks
always quarantine and a cask cannot opt out at the flag level. From **Sept 1, 2026** the *official*
`homebrew-cask` repo disables casks that fail Gatekeeper — but that rule is **official-repo-only; third-party
taps are explicitly exempt.** A cask `postflight` may run `xattr -rd com.apple.quarantine`; the official repo
forbids this, a private tap permits it.

## Decision

Ship corti **unsigned (ad-hoc)** through the shared **private Homebrew tap** (`vasovagal/homebrew-tap`,
which also holds vagus's formula).

- No Apple Developer account, no Developer ID, no notarization. The release pipeline uses Tauri's default
  ad-hoc signing (tauri-action with no `APPLE_*` secrets in the environment).
- The cask (`Casks/corti.rb`, generated by `scripts/render-cask.sh`) carries a `postflight` that strips the
  quarantine xattr, so Gatekeeper does not block first launch.
- **CI never writes the tap.** The release workflow only builds + publishes the GitHub release artifacts.
  After a release, a human regenerates `Casks/corti.rb` via `scripts/render-cask.sh` and commits it to
  `vasovagal/homebrew-tap` (no `HOMEBREW_TAP_TOKEN`, no cross-repo push). A breadcrumb in `CLAUDE.md`
  records the step. (Earlier revisions auto-pushed to a per-app `homebrew-corti` repo via a PAT; dropped in
  favor of one shared, manually-updated tap.)
- The release is tag-triggered and obeys the build/release LAWS: single native macOS runner (§1), no
  PR-matrix re-run on the tag (§3), version-in-tag with a sync gate (§7), one workflow per release event
  (§18), re-run-safe (§19).

## Consequences

- **Free and immediate.** No Apple Program, no cert/secret management, no notarytool. Install is
  `brew tap vasovagal/tap && brew install --cask corti`.
- **TCC re-prompts on every update.** Because the ad-hoc cdhash changes each release, macOS re-asks for
  Microphone + System Audio Recording after each `brew upgrade --cask corti`. Documented in the tap README;
  not a bug.
- **We deliberately bypass Gatekeeper** via the postflight quarantine strip. This is a real security
  tradeoff and is only acceptable for a tap the author controls.
- **macOS floor (latest + one back).** Per ADR 0002 the support floor is `minimumSystemVersion = 15.0`
  (Sequoia); the cask's per-major `depends_on macos: ">= :sequoia"` (15.x) lines up exactly, so there is no
  install-vs-launch gap. Tightening to latest-only would be `15.0`→`26.0` and `:sequoia`→`:tahoe`.

## Upgrade trigger (when to notarize)

Move to Developer ID + notarization (closing the deferred half of issue #23) when any of: corti is
distributed beyond the author's own machines; the per-update TCC re-prompt becomes a real annoyance; or the
quarantine-strip postflight is judged unacceptable. The pipeline is structured so this is additive — add the
Apple secrets + Hardened-Runtime entitlements + a notarize/staple step to `release.yml`, drop the
`postflight` from the cask, and switch `depends_on macos` if the floor changes. Nothing else in the release
shape changes.
